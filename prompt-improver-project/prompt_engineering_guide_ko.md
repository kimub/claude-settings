# Anthropic 프롬프트 엔지니어링 핵심 가이드

## 1. 명확성과 직접성 (be-clear-and-direct)
- 역할(Role), 목적(Goal), 출력 형식(Output format)을 명시
- 모호한 지시어 지양 ("잘 해줘" → "500자 이내 요약문으로 작성")
- 긍정적 지시 사용 ("하지 마세요" 대신 "~하세요")
- 평가 기준: 역할 명시(5), 목적 명시(5), 출력형식(5), 제약조건(5) = 20점

## 2. 멀티샷 프롬프팅 (multishot-prompting)
- 입출력 예시를 2개 이상 포함할수록 성능 향상
- 예시는 실제 사용 사례와 유사하게 작성
- 엣지 케이스 예시 포함 권장
- 평가 기준: 예시 없음(0), 1개(7), 2개 이상(12), 엣지케이스 포함(15)

## 3. 사고 과정 유도 (chain-of-thought)
- "단계별로 생각하세요", "먼저 X를 분석한 후 Y를 도출하세요" 등 사용
- 복잡한 태스크일수록 CoT 필수
- 중간 추론 과정을 명시적으로 요청
- 평가 기준: CoT 없음(0), 암시적(7), 명시적(12), 구조화된 CoT(15)

## 4. XML 태그 구조화 (use-xml-tags)
- 입력 데이터, 지시사항, 출력 형식을 XML 태그로 구분
- 태그명은 의미 있고 일관성 있게 사용
- 중첩 태그로 계층 구조 표현
- 평가 기준: 태그 없음(0), 일부 사용(7), 체계적 사용(12), 중첩구조(15)

## 5. 시스템 프롬프트 (system-prompts)
- 역할과 페르소나를 시스템 프롬프트에 설정
- 전역 규칙과 태스크별 지시 분리
- 보안/안전 가이드라인 포함
- 평가 기준: 시스템 프롬프트 설계 적절성 (10점)

## 6. 프롬프트 체이닝 (chain-prompts)
- 복잡한 태스크를 단계별 서브태스크로 분리
- 이전 단계 출력을 다음 단계 입력으로 연결
- 각 단계별 검증 포인트 포함
- 평가 기준: 단일(0~5), 2단계(7), 3단계 이상(10)

## 7. 긴 컨텍스트 처리 (long-context-tips)
- 핵심 정보를 프롬프트 앞/뒤에 배치
- 긴 문서는 요약 후 참조 방식 사용
- 컨텍스트 윈도우 한계 고려한 청크 전략
- 평가 기준: 고려 없음(0), 부분 고려(3), 적절히 고려(5)

## 8. Extended Thinking (extended-thinking-tips)
- 복잡한 추론이 필요한 경우 명시적으로 깊은 사고 요청
- "모든 가능성을 검토하고", "장단점을 철저히 분석" 등 사용
- 결론 도출 전 충분한 사고 과정 유도
- 평가 기준: 해당없음(5), 암시적(7), 명시적 활용(10)

---

## 점수별 판정
- 90~100점: 최적화 완료, 즉시 사용 가능
- 85~89점: 우수, 사용 가능 (목표 기준)
- 70~84점: 양호, 1~2회 추가 개선 권장
- 50~69점: 보통, 주요 항목 개선 필요
- 50점 미만: 전면 재작성 권장
